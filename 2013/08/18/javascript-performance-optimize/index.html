Javascript 前端开发

javascript性能优化

注意作用域

避免全局查询
局部查找比全局查找快
避免使用with
with()会创建自己的作用域，增加作用域链的长度

选择正确的方法

避免不必要的查找，减少复杂度
优化循环

  1. 减值迭代(从高往低循环)
  2. 简化循环体
  3. 简化终止条件
  4. 避免双重解释(避免eval('alert("1")');避免new Function();避免setTimeout第一参数传入字符串)
  5. 其他事项(原生方法比较快;switch比if-else快[把更可能的case放在前面];位运算比逻辑与、逻辑或都要快)

使用Array作为String Buffer
IE在字符串拼接的时候，效率很低；firefox下，Array的join执行效率是拼接的1.4倍
对于多次重复执行，setIntelval效率高于setTimeout

最小语句化

多个变量声明合并
插入迭代值
将name=value[i];i++;合并为：name=value[i++]
使用数组和对象字面量,缩短定义

优化DOM交互

DOM交互最消耗时间
最小化现场更新
使用innerHTML
innerHTML比createElement快，而且比appendChild快
使用事件代理
如果可能，把事件附加到更高层次的地方
注意NodeList
查询NodeList的属性，方法开销很昂贵;另外特别要优化循环查询
