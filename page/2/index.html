Css3-对于不支持的浏览器解决方案

一、 接受差异
在很多情况下，处理不支持的浏览器的最佳方法就是直接接受视觉表现上的差异，这就是渐进增强的核心思想。很多情况下添加额外的兼容方案反而会让访客更加反感，比如在旧式浏览器上添加背景图片，为了实现圆角，添加圆角背景图片，等一些列图片来实现类似css3效果，这样会导致页面加载缓慢，结果得不偿失。
二、为一个属性同时提供非Css3和Css3的值

三、使用Modernizr检测css3支持
Modernizr是一个第三方脚本，它可以检测用户浏览器对css3和html5特性的支持情况，然后会对html元素添加一些类来表明当前浏览器是否支持某些特性，比如“no-multiplebgs”表明浏览器不支持在单一元素上排布多背景图片，而相应的“multiplebgs”则表明浏览器是支持的。

四、使用javascript模拟Css3
javascript通常能够是的不支持的浏览器实现和高级浏览器一样的效果，例如现在已经有很多脚本实现圆角效果了。 html5.js,你只需且必须在页面head元素内调用它就行了(因为IE必须在元素解析前知道这个元素，所以这个js文件不能在页面底部调用）。

滤镜是微软提供的主要用来并且只在IE浏览器中起作用的创建不同样式表。通常是通过独有的属性滤镜或者-ms-filter属性来实现。具体的属性滤镜语法则依赖于你所使用的特定滤镜，但它们通用的格式是一样的：

在IE8以后，这个语法升级成了-ms-filter作为属性名，你也应该用引号来包裹它的属性值。

DropShadow、 Shadow、 Glow和Blur滤镜用来模拟box-shadow和text-shadow.
Gradient滤镜用来模拟RGBA、 HSLA和线性渐变效果
Alpha和BasicImage滤镜用来模拟opaticy效果
Matrix和BasicImage滤镜用来模拟2D变形效果

滤镜的优点是它不需要javascript并且只在IE中有效，当然也有很多缺点。
长度：描述一个滤镜将会花费很多字符。如果一个css文件中使用很多滤镜，这样会明显增大css文件的大小。为了克服这个，需要将这些样式表放在一个单独的文件里面，使用IE条件表达式来引入它。
无法通过验证：包含滤镜的样式表是无法通过验证的。你必须将这些滤镜放在IE-only的文件里面从而保证至少你的主样式表文件是可以验证通过的。
性能：滤镜会让页面的加载速度变慢，并且消耗大量的内存
粗糙的字体：滤镜会关闭windows的ClearType渲染，因此文本显示不再是抗锯齿的，从而看起来很粗糙。
当然还有很多其他的缺陷，就不一一列出了。

python基础知识

算法：
算法是描述如何完成一项任务的方法。实际上，在编写程序的时候，就是要使用计算机能够理解的语言（如python）来准确地描述算法。这类对机器友好的描述就叫做程序。
函数（function）：
python中的函数类似于java中的方法，可以有参数，并且返回值
模块（module）：
模块就是扩展，它可以导入到python中，从而扩展python的功能。在使用模块的时候需要先导入模块。例如： math模块提供了很多有用的数学函数。

如果不想每次都输入模块名.函数名来调用函数，想直接通过函数名来调用,可以这么做。

但是使用from…import…语句，容易造成命名冲突，比如cmath模块中也有个sqrt()方法，这样的话一旦使用此语句，就没法使用普通的sqrt函数了，所以除非真的需要from这个形式的模块导入语句，否则应该坚持使用普通的import
运算符：
对于基本的加减乘除和java都一样，对于乘除python里进行了增强。

注意：Python 不支持 C 语言中的自增 1 和自减 1 运算符,这是因为 + 和 － 也是单目运算符，Python 会将 --n 解释为-(-n) 从而得到 n , 同样 ++n 的结果也是 n.

Mac截图快捷键用法

Mac OS X 内置的截图应用程序Grab。Grab截屏的特点包括可截取带有光标的屏幕，光标样式可以在设置里选择。还可以利用Grab时间延迟功能，也能截取带菜单的截图。

不用开启程序直接快捷键即可截图，常用快捷键：

⌘-⇧-3: 全屏截屏, 保存至桌面;
⌘-⇧-4: 选定屏幕的某一方形区域截屏, 拖动指针改变选定区域;
⌘-⇧-4, 释放, 然后按空格, 点击, 对当前活动窗口截屏, 并保存至桌面;
⌘-⇧-4, 释放, 选定区域, 按住空格, 可以固定选定方块在屏幕上拖动截屏;
⌘-⇧-4, 释放, 选定区域, 按住⌥, 可以在水平或垂直方向以已选区域为中心, 同比例放大或缩小;
⌘-⇧-4, 释放, 选定区域, 按住⇧, 固定x轴y轴选定宽度, 变动y轴或x轴来截屏;
⌘-⇧-4, 释放, 选定区域, 按住⌥-⇧, 上述5/6的组合;
⌘-⇧-4, 释放, 选定区域, 按住⇧-空格, 上述4/6的组合. ctrl+以上3条快捷键任一条, 把截屏保存至剪贴板;

在以上任一条快捷中, 按Esc取消截屏.

10+ HTML Best practices for Web Developer

HTML已经走过了近20的发展历程。从HTML4到XHTML，再到最近十分火热的HTML5，它几乎见证了整个互联网的发展。但是，即便到现在，有很多基础的概念和原则依然需要开发者高度注意。下面，向大家介绍这些应该遵循的开发原则。

1、善用DIV来布局

当开发一个Web页面时，要考虑第一件事就是区分页面重点。将这些内容用DIV标签包含起来，页面的代码会呈现出整洁、缩进良好的风格。

2、将HTML标签和CSS样式表分割开来

好的页面应该将HTML标签和CSS样式表分割开来。这是每一个Web开发者在首次接触Web开发时就应该知道的一条原则。然而，直到今天，仍然有很多开发者没有严格遵循这一原则。

不要在HTML标签里内嵌样式表代码。开发者应该养成习惯，单独建立文件，用以存放CSS样式表。而这也将方便其他开发者在修改你的代码时，能迅速完成工作。
