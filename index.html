Dialogue – 使用蓝牙连接你的手机在Mac上打电话

Dialogue 是Mac下一款很酷的小应用。使用这款软件可以让你用 Mac化身蓝牙耳机，代替 iPhone 进行电话的接收和拨打工作。

首先使用 Dialogue 前需要将iPhone 和 Mac 两端开启蓝牙并配对连接好，然后在Mac下就可以工作了。来电话后，Mac会自动弹出“来电”操作框；通话过程中还能选择“录音”，录音结束后可保存为M4A格式文件到本地硬盘。

使用Dialogue的通话质量还凑合，推荐使用耳机，这样效果更佳， Dialogue能与Mac通讯录联系人完美整合，你可以在Dialogue里直接查找联系人呼出电话，也可以直接输入要拨打的号码直接呼出。

Dialogue可以在OS X Mavericks 10.9下工作。

IE兼容问题总结

javascript兼容

在js中，各个浏览器的基本语法差距并不大,兼容问题出现在各个宿主环境里（就是所谓的的浏览器中），尤其是事件。

  1. ie6+都可以用事件一级模型和事件二级模型来绑定一个事件,在二级模型事件中,现代浏览器用的是addEventListener,而ie6,7,8用的是attachEvent，9,10两者都可以用

  2. 标准浏览器采用事件捕获的方法，而ie6,7,8用 的事件冒泡机制（所谓事件捕获就是由最外层元素至目标元素最里层，事件冒泡就是由目标元素最里层冒泡到最外层），不过现代浏览器可能更加觉得冒泡机制更好，所以在addEventListener最后加了一个参数，表示是否是捕获机制，一般是false

  3. 获取事件对象 标准浏览器回调函数定义e参数,而ie6,7,8使用的是window.event,获取目标元素,标准浏览器用e.target而ie6,7,8用e.srcElement

深入浅出Spring IOC原理

学过spring的应该知道，spring的核心就是ioc和aop了。今天来说一下ioc到底是什么，原理是什么，主要解决了哪些问题。
IOC拟物化
我不知道大家有没有经常网购的习惯，反正我平常消费的银子70%都是花在网购上，感觉网购很方便，可以随便挑，各种东西应有尽有，还送货上门，乐趣无限啊。说到网购，大家仔细想一下，其实，淘宝网就是很大的ioc容器，你要的东西应有尽有，店家把各自的东西挂到上面，就相当于把Bean放入容器的过程，而我们搜索我们想要的东西的过程，就类似于ioc注入的过程，找到了也就是注入成功了，找不到就抛出异常。意思就是：只要满足我们的条件，哪家卖的都无所谓。言外之意，反应到ioc中就是：只要实现的接口，具体注入哪个实现类都是无关紧要的。
IOC概念
我们都知道ioc是控制反转，什么是控制反转呢？个人认为这个概念比较抽象，另一个名称：DI(依赖注入)能更好的解释这个概念。从字面意义上理解，控制反转就是反过来控制。以前我们需要一个对象，都需要自己去new出来这个对象，控制反转之后，也就是说我们不需要主动自己去new这个对象，我们只要直接去接受这个对象就行了，换句话说这个对象被注入进来了，这就是DI的概念。
IOC原理
我们知道，ioc的最大作用就是bean管理和解耦。到底是如何进行解耦的呢？下面我用数学中的有向图进行说明。我们可以把Spring管理的每一个Bean当做一个节点（假设有N个Bean，那么就有N个节点），于是这些Bean加上相互之间表示调用关系的有向边就组成了一个有向图。毫无疑问，有向图的边越多那么平均来说每个顶点之间的耦合度就越大，最大的情况为完全有向图，即任何两个节点之间都有两条有向边，此时每个顶点的度为2（N-1），也就是说每个Bean与其他的N-1个Bean有耦合，而且还是双向耦合。如下图所示：

在编程的时候，我们都是本着高内聚、低耦合的编程方式，但是我们知道，要想达到完全的没有耦合是不可能的。所以能做的就是尽最大可能的保证松耦合。如何能减少类与类直接的耦合呢？一般我们都是用接口来实现。通过让类与类之间，依赖于接口，而不依赖于类，来达到松耦合的目的。但是无论耦合多么“松散”，具体的实现类还是需要new出来的，于是就得考虑这个new的过程应该放在哪里才是最合适的。如果在每个Bean中都去new这个Bean需要的其他Bean，那么耦合的程度又回到了上图的情况，那是没有意义的。所以我们就需要找一个中间人，在中间人那里实现new的过程，然后把new好的实例传递给需要这个实例的对象，这样的话一切问题就迎刃而解了。因此spring ioc容器就借助这个原理实现了bean管理和松耦合的功能。如下图所示：

上图也就是Spring的数学描述，图中很清晰的表明了在使用Spring之后，任意两个节点（Bean）之间的有向线段变成了0（确切的说不完全是0，因为任意两个节点之间还是可以通过容器连接的），唯一和这些Bean有关系的是容器（无论谁需要什么东西都去找容器，尽量避免两个Bean之间直接打交道）也正因为如此Spring才达到了解耦的目的。

javascript中escape,encodeURI,encodeURIComponent函数

三个方法主要用来转码操作，解决地址栏中文传参等问题，但有所不同。

该方法不会对 ASCII 字母和数字进行编码，也不会对这些 ASCII 标点符号进行编码：- _ . ! ~ * ' ( ) 。 该方法的目的是对 URI 进行完整的编码，因此对以下在 URI 中具有特殊含义的 ASCII 标点符号，encodeURI() 函数是不会进行转义的：;/?:@&=+$,#;此时就必须使用encodeURIComponent()进行编码。
1. 传递参数时需要使用encodeURIComponent，这样组合的url才不会被#等特殊字符截断。
2. 进行url跳转时可以整体使用encodeURI
3. js使用数据时可以使用escape
4. escape对0-255以外的unicode值进行编码时输出%u**格式，其它情况下escape，encodeURI，encodeURIComponent编码结果相同。另外使用unescape()对 escape()编码的字符进行解码。但是ECMAScript v3反对使用该方法，应使用decodeURI()和decodeURIComponent()代替
最多使用的应为encodeURIComponent，它是将中文、韩文等特殊字符转换成utf-8格式的url编码，所以如果给后台传递参数需要使用encodeURIComponent时需要后台解码对utf-8支持（form中的编码方式和当前页面编码方式相同）

Css3-对于不支持的浏览器解决方案

一、 接受差异
在很多情况下，处理不支持的浏览器的最佳方法就是直接接受视觉表现上的差异，这就是渐进增强的核心思想。很多情况下添加额外的兼容方案反而会让访客更加反感，比如在旧式浏览器上添加背景图片，为了实现圆角，添加圆角背景图片，等一些列图片来实现类似css3效果，这样会导致页面加载缓慢，结果得不偿失。

滤镜是微软提供的主要用来并且只在IE浏览器中起作用的创建不同样式表。通常是通过独有的属性滤镜或者-ms-filter属性来实现。具体的属性滤镜语法则依赖于你所使用的特定滤镜，但它们通用的格式是一样的：

DropShadow、 Shadow、 Glow和Blur滤镜用来模拟box-shadow和text-shadow.
Gradient滤镜用来模拟RGBA、 HSLA和线性渐变效果
Alpha和BasicImage滤镜用来模拟opaticy效果
Matrix和BasicImage滤镜用来模拟2D变形效果

滤镜的优点是它不需要javascript并且只在IE中有效，当然也有很多缺点。
长度：描述一个滤镜将会花费很多字符。如果一个css文件中使用很多滤镜，这样会明显增大css文件的大小。为了克服这个，需要将这些样式表放在一个单独的文件里面，使用IE条件表达式来引入它。
无法通过验证：包含滤镜的样式表是无法通过验证的。你必须将这些滤镜放在IE-only的文件里面从而保证至少你的主样式表文件是可以验证通过的。
性能：滤镜会让页面的加载速度变慢，并且消耗大量的内存
粗糙的字体：滤镜会关闭windows的ClearType渲染，因此文本显示不再是抗锯齿的，从而看起来很粗糙。
当然还有很多其他的缺陷，就不一一列出了。

函数（function）：
python中的函数类似于java中的方法，可以有参数，并且返回值
模块（module）：
模块就是扩展，它可以导入到python中，从而扩展python的功能。在使用模块的时候需要先导入模块。例如： math模块提供了很多有用的数学函数。

如果不想每次都输入模块名.函数名来调用函数，想直接通过函数名来调用,可以这么做。

但是使用from…import…语句，容易造成命名冲突，比如cmath模块中也有个sqrt()方法，这样的话一旦使用此语句，就没法使用普通的sqrt函数了，所以除非真的需要from这个形式的模块导入语句，否则应该坚持使用普通的import
运算符：

Mac截图快捷键用法

Mac OS X 内置的截图应用程序Grab。Grab截屏的特点包括可截取带有光标的屏幕，光标样式可以在设置里选择。还可以利用Grab时间延迟功能，也能截取带菜单的截图。

不用开启程序直接快捷键即可截图，常用快捷键：

⌘-⇧-3: 全屏截屏, 保存至桌面;
⌘-⇧-4: 选定屏幕的某一方形区域截屏, 拖动指针改变选定区域;
⌘-⇧-4, 释放, 然后按空格, 点击, 对当前活动窗口截屏, 并保存至桌面;
⌘-⇧-4, 释放, 选定区域, 按住空格, 可以固定选定方块在屏幕上拖动截屏;
⌘-⇧-4, 释放, 选定区域, 按住⌥, 可以在水平或垂直方向以已选区域为中心, 同比例放大或缩小;
